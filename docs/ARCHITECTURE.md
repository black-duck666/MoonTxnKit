# 架构说明

## 领域模型

`model.mbt` 定义隔离级别、事务状态、版本值、写意图、点读与前缀读观察、
保存点、冲突和提交结果。

## MVCC 引擎

`engine.mbt` 维护单调逻辑版本、每键版本链和活动快照。事务优先读取自己的待
提交写，否则读取不超过快照版本的最新值。

提交阶段依次执行：

1. 验证事务仍为活动状态；
2. 对写集执行 first-committer-wins；
3. Serializable 模式校验点读集合；
4. 校验所有前缀谓词内是否出现快照后版本；
5. 为整批写分配同一提交版本；
6. 追加 WAL 并应用版本；
7. 关闭活动快照并更新统计。

## 前缀谓词

`scan_prefix` 遍历快照可见键，叠加事务内写集，并按键排序。保存点会复制谓词
观察，局部回滚可恢复扫描前状态。谓词冲突返回实际变化键和版本，便于重试和
审计。

## 恢复与维护

`wal.mbt` 生成逻辑日志与校验值；`recovery.mbt` 在应用前检查校验、版本连续
性和重复版本一致性。`maintenance.mbt` 按最老活动快照计算压缩低水位。

核心不执行 IO。文件、浏览器存储、数据库和刷盘顺序由外部适配器负责。
