# 架构说明

MoonTxnKit 采用五个职责清晰的层次。

## 领域原子计划

`atomic_plan.mbt` 提供面向工作流、规则引擎和内存服务的声明式入口。
`AtomicPlan` 把业务前置条件和整批写集合交给引擎；`Engine::execute` 在同一
Serializable 快照检查条件，再提交所有变更。业务条件不满足返回
`ConditionFailed`，并发验证失败返回 `CommitRejected`，两类失败不会混淆。

## 领域模型

`model.mbt` 定义隔离级别、事务状态、版本值、读写集合、保存点、冲突和提交结果。模型不依赖引擎实现。

## MVCC 引擎

`engine.mbt` 维护单调逻辑版本、每个键的版本链和活跃快照。事务读取自己的待提交写，否则读取不超过快照版本的最新值。

提交分为验证和应用两阶段：

1. 检查事务仍处于 Active；
2. 验证所有写键没有出现快照后的版本；
3. Serializable 模式验证所有已读键；
4. 为整批写分配同一个提交版本；
5. 追加 WAL 后原子地应用版本；
6. 关闭活跃快照并更新统计。

## 恢复

`wal.mbt` 生成逻辑日志和校验值；`recovery.mbt` 先验证完整输入，再应用记录。版本断层、校验失败或重复版本内容不一致都会停止恢复。

## 维护

`maintenance.mbt` 计算最老活跃快照。压缩时保留低水位可见锚点及其后的版本，确保长事务仍可读取原快照。

## 报告

`reporting.mbt` 提供冲突、提交结果、历史版本和校验问题的 JSON 输出，核心不直接执行 IO。
