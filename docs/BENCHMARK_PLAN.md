# 基准与复杂度

## 固定工作负载

- 单事务写入 10,000 个 `inventory:` 键；
- Serializable 前缀扫描并稳定排序；
- 并发插入一个匹配键；
- 扫描事务提交并产生 `PredicateWrite`；
- WAL 全量恢复并运行内部校验。

## 复杂度

- 点读：当前版本链反向扫描，最坏 `O(v)`；
- 前缀扫描：`O(k + r log r + w*r)`，`k` 为键数，`r` 为结果数，`w` 为待写数；
- 谓词提交校验：每个前缀遍历当前键空间；
- WAL 恢复：与日志记录及写操作总数线性相关。

当前实现优先保证语义透明和确定性。后续可在不改变事务 API 的前提下加入前缀
索引与二分版本链。正式性能数字必须记录 MoonBit 版本、后端、硬件和原始命令。
