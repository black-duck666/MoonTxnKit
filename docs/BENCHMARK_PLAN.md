# 基准计划

性能测试不与功能测试混在一起，后续建立固定数据集并记录 MoonBit 版本、目标后端和输入规模。

## 场景

1. 单键连续提交：衡量版本链追加成本。
2. 读多写少：固定 10000 个键，比较 Snapshot 与 Serializable 的提交验证开销。
3. 热点冲突：多个事务写同一键，统计拒绝率和验证吞吐。
4. WAL 恢复：重放 1000、10000、100000 条记录，记录每秒应用数。
5. 压缩：不同活跃快照深度下统计回收版本数量和耗时。

## 指标

- 每秒事务数；
- P50/P95 单次提交时间；
- 每事务读写集合大小；
- WAL 字节数和重放速度；
- 压缩前后版本数量。

当前阶段不填写未经实际运行的数据。
