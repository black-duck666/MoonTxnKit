# 隔离语义

## Snapshot

事务固定开始时的逻辑版本。重复点读和前缀扫描不会看到后续提交；提交时只检查
写键是否在快照后被修改。

## Serializable

在 Snapshot 基础上增加：

- 点读集合校验：已读取键在快照后发生变化则拒绝；
- 前缀谓词校验：匹配前缀的任意键在快照后新增、修改或删除则拒绝。

这能防止常见的写偏斜和前缀扫描幻读。例如两个事务同时扫描
`slot:available:` 为空并各自创建预订，只允许先提交者成功。

## 明确边界

当前谓词是字符串前缀，不支持任意范围比较、复合索引、SQL 谓词或完整 SSI
依赖图。因此项目只声明覆盖点读和前缀键空间，不把实现描述为通用数据库级
Serializable。
