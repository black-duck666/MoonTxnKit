# WAL 与恢复保证

## 日志内容

每个非只读提交产生一条 `WalRecord`，包含事务编号、事务快照、提交版本、写操作列表和确定性校验值。删除以 `value = None` 表示。

## 顺序

引擎在应用版本链之前把记录追加到内存 WAL。外层持久化适配器应在确认日志持久化后再向业务返回成功；当前核心不实现文件刷盘。

## 重放规则

- 校验值错误：整次重放在应用前失败；
- 提交版本断层：拒绝重放；
- 已存在的同版本同校验记录：幂等跳过；
- 已存在的同版本不同校验记录：拒绝重放；
- 成功记录按提交版本依次重建版本链。

## 非目标

当前版本不负责文件格式、分段日志、fsync、校验和算法的密码学安全、跨进程锁或副本一致性。这些能力应位于后续适配包或上层系统。
